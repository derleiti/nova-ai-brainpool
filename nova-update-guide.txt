# Nova AI Brainpool Update Guide

## 🔄 Update-Übersicht

Diese Anleitung führt Sie durch den Update-Prozess von Nova AI Brainpool und erklärt wichtige Änderungen zwischen den Versionen.

## 📋 Vor dem Update

### 1. Backup erstellen
```bash
# Vollständiges Backup
wp db export nova-ai-backup-$(date +%Y%m%d).sql
tar -czf nova-ai-files-backup-$(date +%Y%m%d).tar.gz \
  wp-content/plugins/nova-ai-brainpool/ \
  wp-content/uploads/nova-ai-images/
```

### 2. Systemanforderungen prüfen
```bash
# PHP Version prüfen
php -v

# WordPress Version prüfen
wp core version

# Speicherplatz prüfen
df -h
```

### 3. Plugin-Kompatibilität
- Deaktivieren Sie Caching-Plugins temporär
- Prüfen Sie Konflikte mit anderen AI-Plugins
- Stellen Sie sicher, dass keine kritischen Operationen laufen

## 🚀 Update-Methoden

### Methode 1: WordPress Admin (Empfohlen)
```
WordPress Admin → Dashboard → Updates → Plugin Updates
oder
WordPress Admin → Plugins → Nova AI Brainpool → Update verfügbar
```

### Methode 2: Automatische Updates
```php
// wp-config.php - Automatische Plugin-Updates aktivieren
define('WP_AUTO_UPDATE_CORE', true);
add_filter('auto_update_plugin', '__return_true');
```

### Methode 3: Manuelle Installation
```bash
# Plugin deaktivieren
wp plugin deactivate nova-ai-brainpool

# Neue Version hochladen
wp plugin install nova-ai-brainpool.zip --force

# Plugin aktivieren
wp plugin activate nova-ai-brainpool
```

### Methode 4: WP-CLI
```bash
# Update über WP-CLI
wp plugin update nova-ai-brainpool

# Alle Plugin-Updates
wp plugin update --all
```

## 🔄 Migration zwischen Versionen

### Von 1.x zu 2.0

#### 1. Datenbank-Migration
```sql
-- Automatische Migration beim Update
-- Neue Tabellen werden erstellt:
-- wp_nova_ai_crawled_content
-- wp_nova_ai_generated_images
-- wp_nova_ai_network_nodes (NovaNet)
```

#### 2. Konfiguration migrieren
```bash
# Alte Einstellungen werden automatisch übertragen
# Neue .env Datei konfigurieren
cp .env.example .env
nano .env
```

#### 3. API-Konfiguration aktualisieren
```php
// Alte API-Keys bleiben erhalten
// Neue Provider hinzufügen nach Bedarf
WordPress Admin → Nova AI → Settings → AI Provider
```

### Von Beta-Versionen

#### Beta zu Stable Migration
```bash
# Backup der Beta-Konfiguration
wp option get nova_ai_settings > beta-config-backup.json

# Clean Installation der Stable Version
wp plugin deactivate nova-ai-brainpool
wp plugin delete nova-ai-brainpool
wp plugin install nova-ai-brainpool.zip
wp plugin activate nova-ai-brainpool

# Konfiguration wiederherstellen
# (Manuelle Überprüfung empfohlen)
```

## ⚠️ Troubleshooting

### Häufige Update-Probleme

#### Problem: Plugin lässt sich nicht aktivieren
```bash
# Lösung 1: Berechtigungen prüfen
chmod -R 755 wp-content/plugins/nova-ai-brainpool/

# Lösung 2: Speicher erhöhen
ini_set('memory_limit', '512M');

# Lösung 3: Manuell aktivieren
wp plugin activate nova-ai-brainpool
```

#### Problem: Datenbankfehler nach Update
```sql
-- Lösung: Tabellen manuell reparieren
REPAIR TABLE wp_nova_ai_conversations;
REPAIR TABLE wp_nova_ai_messages;
REPAIR TABLE wp_nova_ai_crawled_content;
REPAIR TABLE wp_nova_ai_generated_images;
```

#### Problem: API-Verbindungsfehler
```bash
# Lösung 1: SSL-Zertifikate prüfen
curl -I https://ailinux.me/api/v1

# Lösung 2: Firewall-Regeln prüfen
# Lösung 3: API-Schlüssel neu eingeben
```

#### Problem: Crawler funktioniert nicht
```bash
# Lösung 1: WP-Cron prüfen
wp cron event list

# Lösung 2: Cron-Events neu registrieren
wp cron event schedule nova_ai_auto_crawl now hourly

# Lösung 3: Manuelle Crawler-Ausführung
wp cron event run nova_ai_auto_crawl
```

#### Problem: Bilder werden nicht generiert
```bash
# Lösung 1: API-URL prüfen
curl -I https://ailinux.me:7860

# Lösung 2: Upload-Verzeichnis prüfen
ls -la wp-content/uploads/nova-ai-images/
chmod 755 wp-content/uploads/nova-ai-images/

# Lösung 3: PHP-Memory erhöhen
ini_set('memory_limit', '512M');
```

### Debug-Modus aktivieren

```bash
# .env Datei
NOVA_AI_DEBUG=true
WP_DEBUG=true
WP_DEBUG_LOG=true
WP_DEBUG_DISPLAY=false

# Log-Datei überwachen
tail -f wp-content/debug.log | grep "Nova AI"
```

## 📊 Update-Überwachung

```php

        // Post-Update-Aktionen
```
